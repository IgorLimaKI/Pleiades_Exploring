O que é o Pleiades?

Pleiades é uma comunidade responsável por manter diariamente um “dicionário geográfico” de locais antigos, como templos, fortes, cavernas e outras estruturas consideradas portadoras de valor histórico. Esta comunidade possui uma database de tamanho considerável, contendo informações como localização, descrição, nome, período relacionado e conexões de mais de 37 mil locais, a maior parte deles presente no continente europeu e na porção norte do continente africano.

Devido ao volume de dados, é um desafio considerável traduzir o conteúdo reunido por essa comunidade em uma linguagem visual que seja agradável e interessante para o público comum, como pode ser observado no projeto de Ryan M.Horne de observar os locais e suas conexões espalhados pelo planeta. Portanto, o objetivo desta página é explorar alguns dos aspectos dos dados de Pleiades, fornecer algumas representações visuais de parte de seu conteúdo e permitir que seus leitores possam interagir com as descobertas apresentadas.

No mapa abaixo, cada ponto representa um local catalogado. Você pode obter mais informações sobre cada local clicando sobre os pontos, manipulando o zoom ou arrastando o mapa para os lados para ter melhor noção da localização de cada um.

Dados Utilizados

O conjunto de dados usado para a geração dos gráficos é composto originalmente por 3 arquivos CSV (UTF-8) separados, pleiades-locations (18.5 MB), pleiades-places (18.5 MB) e pleiades-names (12.5 MB), obtidos na página de downloads da comunidade no dia 08/06/2020 07:10 (BRT). Eles foram filtrados e organizados em dois arquivos finais: um CSV (9.81 MB) contendo os dados gerais de cada local e um JSON (832 KB) contendo as algumas conexões registradas entre eles. Dos mais de 37 mil registros, 29.767 foram considerados completos e relevantes o suficiente para serem utilizados como base para o painel interativo e apenas 6.534 válidos para se apresentar as conexões.

-- -- -- -- --

Os dados que foram removidos não possuíam latitude ou longitude, o que os tornava dispensáveis já que uma das principais visualizações desse projeto é o mapa, ou se tratavam de repetições de outros locais. Entre as repetições, os dados de cada atributo disponível foram combinados para se obter um único registo com a maior quantidade de informações possível. O processo completo de limpeza e combinação de arquivos, incluindo os arquivos originais, intermediários e resultantes do processo, podem ser encontrados no repositório do nosso projeto na pasta PleiadesDatasetCleaning.

Dificuldades e Aprendizado

Como foi inicialmente comentado, é desafiador trabalhar com visualizações variadas de uma quantidade volumosa de dados, principalmente no que se refere ao seu preparamento e sua organização, tanto para fornecer as informações com a maior qualidade possível quanto para evitar poluição e desconforto visual para o público. Entretanto, é extremamente prazeroso ir descobrindo aos poucos o mundo escondido nesses arquivos e recompensador apresentar este mundo graficamente para outras pessoas.

Um grande desafio foi a pouca experiencia em desenvolvimento web e javascript, o que dificultou o desenvolvimento do site e a migração dos protótipos do observable, alem disso, fazer a interação entre o gráfico de bolha e os demais se mostrou mais complicado por não ser um chart do DC.js, tendo que ter sido feita manualmente. Essa experiência foi uma ótima oportunidade de nos aprofundar nas bibliotecas D3, DC e Crossfilter e entender melhor as dificuldades de criar visualizações de dados que atinjam seu objetivo.
